# 国际化语言

为了方便开发多语言应用，框架内置了国际化（i18n）支持。

# 准备

在项目根目录的 .hbuilderx 目录下新建 i18n 目录，在 i18n 目录下以各国语言标识（例如：简体中文（中国）：zh_CN）为js文件名，例如：新建zh_CN.js，为简体中文。

# 常用国际化的语言标识

# zh-CN.js示例

简体中文(中国)，JS文件格式需一致，内容为各国语言。

module.exports = {
	tabbar: {
		home: '首页',
		me: '我的'
	}
}

# en-US.js示例

英语(美国)，JS文件格式需一致，内容为各国语言。

module.exports = {
	tabbar: {
		home: 'Home',
		me: 'Me'
	}
}

# 页面使用

页面中使用通过vuex状态管理state进行动态绑定显示。

<template>
	<view>
		<text>tabbar.home</text>
	</view>
</template>

<script>
	import {
		mapState
	} from 'vuex'
	export default {
		data() {
			return {

			}
		},
		computed: {
			...mapState({
				// 箭头函数可使代码更简练
				tabbar: state => state.i18n.pack.tabbar
			})
		}
	}
</script>

# 切换语言

通过调用 this.$store.commit('i18n/i18nLanguage','各国语言标识(对应的i18n目录下js文件名)') 方法切换国际化语言。

this.$store.commit('i18n/i18nLanguage','zh-CN')
